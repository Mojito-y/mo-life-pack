## Why

做一级市场 FA 或行业项目初筛时，不同行业真正重要的判断词条完全不同：医疗器械看注册路径、临床证据和入院支付；AI 应用看数据闭环、交付形态和客户留存；半导体看制程、良率、供应链和国产替代；消费品牌看渠道、复购、毛利和内容投放。当前痛点不是“缺少一个固定模板”，而是缺少一个能让用户按行业自定义关键词条，并把散乱项目材料转成可判断、可追问、可推进的初筛 DD 卡的工作流。

这个 change 的目标是把原先“医疗器械/脑机接口 DD Agent”升级为“行业可配置 DD Agent”：飞书机器人 agent 负责资料接入、证据抽取、按行业词条初筛、输出卡片和生成后续尽调动作。

## What Changes

- 新增一个行业可配置 DD Agent，而不是限制在医疗器械/脑机接口。
- 调整一键安装和初始化流程：第一步先让用户选择要安装的 agent，并展示每个 agent 的中文用途说明。
- 引入行业 profile 机制，允许每个行业定义自己的关键词条、术语别名、字段解释、证据要求、红旗项、评分维度和输出结构。
- 支持从飞书消息、粘贴文本、上传资料和链接中收集项目材料。
- 根据用户选择的行业 profile 生成一页 FA 可用的项目初筛 DD 卡，覆盖基础信息、行业关键词条、证据状态、风险判断和下一步动作。
- 对事实、推断、未知项和 unsupported 项做显式区分，避免把材料里没有证据的信息写成确定结论。
- 提供医疗器械/脑机接口作为默认行业 profile 示例，但实现上不把该行业写死。
- 输出可执行的后续 DD 清单，包括需要公司补充的材料、需要访谈的专家类型、建议追问的问题。

## Capabilities

### New Capabilities
- `industry-dd-agent`: 根据用户选择或自定义的行业 profile，把项目材料生成行业化 FA 初筛 DD 卡。

### Modified Capabilities
- None.

## Impact

- 新增一个 OpenSpec capability，用于约束行业可配置 DD Agent 的行为和输出。
- 后续实现预计会涉及一键安装向导、agent catalog、行业 profile 配置、飞书机器人命令入口、资料解析、项目本地记忆、证据抽取、行业词条评估模板、卡片/Markdown 输出。
- 不改变现有 Mo Coach 行为，不引入破坏性变更。
